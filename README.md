# チーム３　チームワーク課題：簡易ToDoアプリ
<img width="1408" height="903" alt="image" src="https://github.com/user-attachments/assets/9150acaf-20a8-47ca-a22d-5d2110250d18" /><br/>
<br/>
本システムは、チーム進捗などのタスクを名称・優先度・期日で管理するためのWEBアプリケーションです。  
タスクを一覧で表示し、タスクの状態を未完了・完了別に管理することができます。

## チームメンバ
- 渋谷　
- 福山　
- 越智　孝洋

# 機能

## 基本機能
- タスクを一覧で表示する
- タスクの状態（「すべて」・「未完了」・「完了」）でフィルター表示
- 1件のタスクに対し、タスク名・期限・詳細内容・優先度を登録
- 既存のタスクを編集・更新・削除が可能
- 一覧画面上でワンクリックでタスクの完了⇔未完了の切り替えが可能

## 追加の機能
- タスクの優先度別に色分けをして表示
- タスクの期日を昇順で表示する（期日が近いものが一番上に表示）
- タスクにディレイがあると、期日の横にアラート表示

## タスクの登録
1. 【タスク一覧】画面から「タスク作成ボタン」をクリックします。
2. 【タスクの新規作成】画面で登録したいタスクを入力します。
    - タスク名　：　30文字以下でタスクの名称を記載します。<font color="Red">*必須</font>
    - 優先度　　：　高・中・低でタスクの優先度を設定します。
    - 期限　　　：　タスクの期限を入力します。入力欄右側のカレンダーアイコンで、GUI入力も可能です。*必須
    - 詳細内容　：　タスクの詳細事項を記入します。
3. 【タスクの新規作成】画面内の「タスク作成ボタン」をクリックします。

## タスク表示の切り替え（フィルター）
1. 【タスク一覧】画面でフィルターをクリックし、プルダウンメニューを開きます。
2. 表示したいタスクの一覧を選択します。

## タスク状態の切り替え
1. 【タスク一覧】画面で任意のタスクの、タスク完了ボタンをクリックします。

## タスクの編集
1. 【タスク一覧】画面で任意のタスクをクリックします。
2. 【タスクの詳細】画面で、「タスク編集ボタン」をクリックします。
3. 【タスクの編集】画面で、タスクの内容を修正します。
4. 【タスクの編集】画面の「タスク更新ボタン」をクリックします。

## タスクの削除
1. 【タスク一覧】画面で任意のタスクをクリックします。
2. 【タスクの詳細】画面で、「タスク削除ボタン」をクリックします。

# 表示・機能上のアピールポイント
## タスク一覧をカード状にシンプル表示
【タスク一覧】画面は、完了ボタン・タスク名・期限だけの表示として、シンプルに構成しました。  
優先度は、高（赤）→中（黄）→低（緑）で色分け表示し、直感的に分かりやすいようにしました。  
また、完了タスクについても、タスク色の変更や打消し線などで、完了済みであることを分かりやすく表示しています。

## 詳細画面
【タスク一覧】画面でタスクをクリックすることで、詳細画面が表示されます。  
一覧には表示されていない、タスクの詳細内容などはこちらから確認でき、そのまま編集画面に遷移することが可能です。  
<br/>
【タスクの詳細】画面の表示中は、画面背景（タスク一覧）を暗くすることで表示を見やすくしています。  
また、画面背景部をクリックすることで、「閉じるボタン」を押さずとも【タスクの詳細】画面を非表示にすることが可能です。

## アラート表示
未完了のタスクの期限が、過去になっている場合は、期限の横にアラートマークを表示します。

## スクロールボタンの実装
タスクの表示が画面を見切れるとき、一定以上の画面スクロールを行うとTOPに戻るボタンが表示されます。

## レスポンシブ対応
スマホやPCなど、画面サイズが変わっても、表示が大きく崩れないようにしました。  
例えば、画面サイズが一定以下になるとフィルターが2行目に表示されます。 

# 実装上のアピールポイント
## 【タスクの新規作成】画面と【タスクの編集】画面の共通化
両者の画面は、共通のソース（ToDoEditor）をベースに、起動時のタスクIDの有無で、一部文言や処理内容を切り替えて動作します。
これにより、画面の統一感が自動的に担保され、また修正時のコストが最小限になります。

## フィルタ切替後のリダイレクト
フィルターは、条件を変えてのSQLからデータ再取得を実行するため、POSTで操作をします。  
このとき、再表示をOnPost内で行わず、リダイレクトをすることで、OnGetが走り再表示が実行されます。
また、ユーザーがブラウザをリロードしても、OnGetが再実行されるだけなので、再送信ダイアログなども表示されません。
