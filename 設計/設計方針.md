## 画面・C#クラス・インターフェース定義、アプリケーション設定

- Program.cs
  - インターフェースの定義などを行う

- _Layout.cshtml
  - アプリの外見を決める

- Priorities.cs
  - 優先度設定を表すレコード
  - prioritiesテーブルに対応
  - int Id　・・・　優先度ID
  - int Name　・・・　優先度ラベル

- Tasks.cs
  - タスク1件を表すレコード
  - tasksテーブル中の1件レコード（1行）に対応
  - int TaskId　・・・　タスクのID
  - string TaskName　・・・　タスク名
  - string ContentText　・・・　タスクの詳細内容
  - DateTime DeadlineAt　・・・　締切日
  - int PriorityId　・・・　優先度ID
  - DateTime CompletedAt　・・・　完了日
  
- Index.cshtml / Index.cshtml.cs
  - アプリのホーム画面、タスク一覧の表示、フィルタリング、詳細表示を行う
  - タスクの編集・新規作成画面を呼び出す
  - List<Tasks> TaskList　・・・　タスクの一覧
  - List<Priorities> PriorityList　・・・　優先度の選択肢
  - List<SelectListItem> FilterList　・・・　フィルターの選択肢
  - ListFilterMode SelectedFilter 　・・・　フィルターの選択状態、初期値は未完了とする
  - IActionResult OnGet()　・・・　優先度の初期化、タスク一覧の初期化を行う
  - IActionResult OnPostDelete() ・・・　指定したタスクのDeletedAtを更新して、Indexにリダイレクトする
  - IActionResult OnPostComplete()　・・・　指定したタスクのCompletedAtを更新して、Indexにリダイレクトする　
  
- ToDoEditor.cshtml / ToDoEditor.cshtml.cs
  - タスクの編集、新規作成を行う
  - List<Priorities> Options　・・・　優先度の選択肢
  - string? TaskName　・・・　タスクの名称
  - int SelectedProperty　・・・　優先度
  - DateTime? DeadlineAt　・・・　期限
  - string? ContentText　・・・　詳細内容
  - string FormTitle　・・・　フォームのタイトル、初期値は新規作成
  - string PostButtonName　・・・　フォームボタンの名称、初期値は作成
  - IActionResult OnGet(string? id)　・・・　id省略なら新規作成、id指定されたら編集モードで起動、編集モードではタイトルを編集、ボタンを更新に書き換える
  - IActionResult OnPost(int? id)　・・・　id省略なら新規作成、id指定されたら編集モードでDBに登録または更新を行い、Indexにリダイレクトする

- ListFilterMode.cs
  - フィルターの設定を表すenum
  - Continue = 0, Complete, All

- ITaskDataEditor.cs
  - タスクデータを編集・管理するインターフェース
  - List<Properties> GetPriorityList()　・・・　Propertiesのリストを取得する
  - List<Tasks> GetTaskList(ListFilterMode mode)　・・・　modeの条件に従って、Tasksのリストを取得する
  - Tasks? GetTask(int taskId)　・・・　指定したタスクIDのTasksデータを取得する
  - void Add(Tasks newTask)　・・・　引数に渡されたnewTaskを追加する
  - void Update(Tasks task)　・・・　引数に渡されたtargetTaskを更新する
  - void Delete(int taskId)　・・・　指定されたtaskIdのタスクを削除する

- TaskDataEditor.cs
  - ITaskDataEditorの実装
  - List<Properties> GetPriorityList()　・・・　propertiesテーブルからSELECTして、Propertiesのリストを返す
  - List<Tasks> GetTaskList(ListFilterMode mode)　・・・　modeの条件に従ってtasksテーブルからSELECTし、Tasksのリストを返す
  - Tasks? GetTask(int taskId)　・・・　指定したタスクIDのTasksデータを、tasksテーブルからSELECTして返す
  - void Add(Tasks newTask)　・・・　引数に渡されたnewTaskを、tasksテーブルに追加する
  - void Update(Tasks task)　・・・　引数に渡されたtargetTaskと、同じタスクIDをもつレコードを、targetTaskで更新する
  - void Delete(int taskId)　・・・　指定されたtaskIdのタスクの、deleted_atに日付を入れて更新する（論理削除の実行）

## Razor Pages制御文
- Index.cshtml
  - @foreach(var task in TaskList)　・・・　TaskListを1行づつに分解し、Task1件の定義を行う
  - @switch(task.PriorityId)　・・・　タスクの優先度に応じて、タスク1件ごとの色を変える

- ToDoEditor.cshtml
  - @page "/edit/{id?}"　・・・　ページURLを"/edit"に変える、省略可能なidをURLとして受け取れるようにする（例：/edit/123）
  - @foreach(var priority in PriorityList)　・・・　PriorityListを1行づつに分解し、優先度のドロップダウンリストの定義を行う

## JavaScript機能・定義

### 主な機能

- Index.cshtml（メイン側）
  - @Html.Raw(System.Text.JsonSerializer.Serialize(Model.TaskList))　・・・　TaskListをIndexModelからJson形式で受け取る、プロパティはcamelCaseになる
  - @Html.Raw(System.Text.JsonSerializer.Serialize(Model.PropertyList))　・・・　PrioritiesをIndexModelからJson形式で受け取る
  - 詳細メニュー表示　・・・　タスク1件を押すと.activeで開閉、タスク1件ごとのdata-id="@task.taskId"を振り、これをつかって、@Model.TaskListから必要なタスクを詳細画面に渡す
  - 完了したタスク　・・・　task.completedAtで完了時の見た目に変える
  - トップに戻るボタン　・・・　スクロール量で.active切替で表示、#に飛ぶ
  - 以下はチャレンジ課題の領域なので、追加実装で想定しておく
    - 遅れマーク表示　・・・　タスク1件ごとにtask.deadlineAtと今日の日付を比較して判定、.activeの有無で表示
    - ~~ タスクの残り時間/遅れ時間表示　・・・　比較した差分を表示、0以上なら”残り”、0未満なら"遅れ"に変える、色変更も行う ~~
      
- Index.cshtml（詳細メニュー側）
  - 詳細メニュー表示中は、メニュー外暗転＆暗転部クリックメニュー閉じる
  - ×ボタン　・・・　自身の.activeを切り替える
  - 優先度　・・・　task.priorityIdに応じて、”優先度：＊”の表記と書式を切り替える
  - priority　・・・　優先度表示する文字列
  - taskName　・・・　タスク名を表す文字列
  - deadlineAt　・・・　締切日を表す文字列
  - contentText　・・・　詳細内容を表す文字列
  - 編集ボタン　・・・　詳細メニュー表示時にhrefを /ToDoEditor/**　のように書き換える
  - 以下はJSと直接の関係は薄いが表記
    - 完了ボタン　・・・　SelectedIDを完了状態にする（Post発行）
    - 削除ボタン　・・・　SelectedIDを削除状態にする（Post発行）
